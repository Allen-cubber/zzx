第 1 章 概述#

1. 说明互联网边缘部分和核心部分的作用。#

互联网由边缘部分和核心部分组成。边缘部分由连接在互联网上的主机或端系统组成，主要运行各种网络应用，为用户提供通信和资源访问服务，常见通信方式有客户/服务器方式（C/S）和对等方式（P2P）。核心部分由大量路由器和通信链路组成，主要作用是为边缘主机提供连通性，并负责分组的路由选择和转发。互联网核心采用分组交换技术，即把报文划分成较小的分组，每个分组带有首部，在路由器中采用存储转发方式逐跳转发到目的主机。

2. 计算机网络的性能指标有哪些？#

计算机网络的主要性能指标有速率、带宽、吞吐量、时延、时延带宽积、往返时间 RTT 和利用率。其中速率和带宽反映网络传输能力，吞吐量反映实际传输效果，时延和 RTT 反映传输所需时间，时延带宽积反映链路中可容纳的数据量，利用率反映网络资源使用程度。

3. 什么是计算机网络分层结构体系？#

计算机网络分层结构体系是将网络通信功能划分为若干层次，每一层利用下一层提供的服务，完成本层功能，并向上一层提供服务。同层实体之间通信所遵守的规则称为协议，协议规定通信的语法、语义和同步；下层向上层提供的功能称为服务。分层结构使网络系统层次清楚、各层相对独立，便于实现、维护和扩展。

第 2 章 物理层#

1. 物理层的任务是什么？#

物理层的主要任务是在物理传输媒体上透明地传输比特流，规定通信接口的机械、电气、功能和过程特性，为数据链路层提供比特传输服务。

2. 列举几种常见的信道复用技术。#

常见的信道复用技术有频分复用、时分复用、统计时分复用、波分复用和码分复用。

3. 列举几种常见的宽带接入技术。#

常见的宽带接入技术有 ADSL（普通电话线接入，上下行速率不对称）、FTTx（光纤接入）、HFC 接入（混合光纤同轴电缆接入）、以太网接入、无线宽带接入和卫星宽带接入等。

第 3 章 数据链路层#

1. 数据链路层的点对点信道和广播信道的特点。#

数据链路层信道可分为点对点信道和广播信道。点对点信道只连接两个节点，通信双方明确，不存在多个站点竞争信道的问题，常用协议有 PPP、HDLC。广播信道由多个站点共享通信介质，一个站点发送的数据其他站点都可能收到，因此需要介质访问控制协议来协调发送，如以太网中的 CSMA/CD、无线局域网中的 CSMA/CA，并通过 MAC 地址识别接收方。

2. 什么是数据链路层的三个基本问题？#

数据链路层的三个基本问题是封装成帧、透明传输和差错检测。封装成帧是在网络层数据前后加上首部和尾部，形成帧，并实现帧定界；透明传输是指无论数据中出现什么比特组合，都能不受控制字符影响而正确传输；差错检测是通过校验方法如 CRC 检测帧在传输过程中是否发生差错。

3. 什么是以太网mac层的硬件地址？#

以太网 MAC 层的硬件地址即 MAC 地址，长度为 48 位，用十六进制表示。它用于在局域网中唯一标识网络接口，MAC 帧首部中包含目的 MAC 地址和源 MAC 地址。交换机根据目的 MAC 地址转发帧。广播 MAC 地址为 FF-FF-FF-FF-FF-FF。MAC 地址属于数据链路层地址，而 IP 地址属于网络层地址，二者通常通过 ARP 协议建立映射关系。

4. 适配器、转发器、集线器、网桥、以太网交换机的作用和使用场合#

设备工作层次主要作用是否识别 MAC 地址使用场合
适配器/网卡物理层 + 数据链路层主机接入网络是电脑、服务器接入网络
转发器物理层再生信号，延长距离否延长传输距离
集线器物理层多端口信号转发器否早期连接多台主机
网桥数据链路层连接网段，过滤帧是连接两个局域网网段
以太网交换机数据链路层多端口高速网桥是现代局域网连接设备

第 4 章 网络层#

1. 什么是虚拟互联网络和两种服务、两个层面的概念？#

虚拟互联网络：用 IP 协议把不同的物理网络互连起来，对上层表现为一个统一的网络。
两种服务：虚电路服务和数据报服务。互联网采用数据报服务。
数据报服务特点：无连接、尽最大努力交付、不保证可靠性。
两个层面：数据平面负责分组转发，控制平面负责路由选择。

2. 说明IP地址和MAC地址之间的关系。#

IP 地址是网络层地址，用于逻辑寻址和路由选择；MAC 地址是数据链路层地址，用于局域网内帧的传输。IP 地址标识最终通信对象，MAC 地址标识当前链路上的下一跳设备。IP 地址到 MAC 地址的映射通常通过 ARP 协议完成。

3. 区分传统分类的IP地址和CIDR。#

传统分类 IP 地址把 IPv4 地址分为 A、B、C、D、E 类，其中 A、B、C 类用于单播通信，默认掩码分别为 255.0.0.0、255.255.0.0、255.255.255.0。
CIDR 是无分类编址方法，用 “IP 地址/前缀长度” 表示网络，打破了 A、B、C 类固定边界，提高了 IP 地址利用率，并支持路由聚合。

4. 说明路由选择协议的工作原理。#

路由选择协议通过路由器之间交换路由信息，动态了解网络拓扑和各网络的可达情况，并根据一定的度量标准和路由算法计算到目的网络的最佳路径，形成和更新路由表。路由器转发 IP 分组时，根据路由表和最长前缀匹配原则选择下一跳或输出接口。常见路由选择协议有 RIP、OSPF 和 BGP，其中 RIP 采用距离向量算法，OSPF 采用链路状态算法，BGP 采用路径向量算法。

第 5 章 运输层#

1. 说明运输层为相互通信的应用进程提供逻辑通信。#

运输层位于网络层之上，它利用网络层提供的主机到主机通信服务，为运行在不同主机上的应用进程之间提供端到端的逻辑通信。运输层通过端口号标识不同的应用进程，使应用进程看起来好像直接通信。TCP 和 UDP 是运输层的两个主要协议。

2. 端口和套接字的含义是什么？#

端口号用于标识主机中的应用进程；套接字是 IP 地址和端口号组成的通信端点。运输层通过端口号实现进程到进程的通信，通过套接字唯一标识网络通信的一端。一条 TCP 连接需要由通信双方共同确定。

3. UDP和TCP的特点分别是什么？#

UDP 是无连接、不可靠、面向报文的运输层协议，首部开销小、传输效率高，适合实时应用。TCP 是面向连接、可靠、面向字节流的运输层协议，通过确认、重传、流量控制和拥塞控制等机制保证可靠传输，适合对可靠性要求高的应用。

4. 在不可靠网络上实现可靠传输的工作原理是什么？#

在不可靠网络上实现可靠传输，通常采用差错检测、序号、确认和重传机制。发送方给数据分组编号并发送，接收方通过校验检测差错，正确收到后返回确认 ACK；若发送方超时未收到确认，则重传相应分组。序号可用于识别丢失、重复和乱序分组，滑动窗口机制可提高传输效率。TCP 的可靠传输就是基于这些机制实现的。

5. 停止等待协议和ARQ协议是什么？#

ARQ （Automatic Repeat reQuest）是自动重传请求协议，通过差错检测、确认和超时重传实现可靠传输。停止等待协议是指发送方每发送一个分组后就停止发送，等待接收方确认，收到确认后才发送下一个分组。停止等待 ARQ 是最简单的 ARQ 协议，优点是简单可靠，缺点是信道利用率低。此外还有连续 ARQ 协议等。

6. 说明TCP的滑动窗口、流量控制、拥塞控制和连接管理。#

TCP 通过滑动窗口机制连续发送多个字节并根据确认号移动窗口，提高传输效率；通过接收窗口 rwnd 实现流量控制，防止发送方发送过快导致接收方缓冲区溢出；通过拥塞窗口 cwnd、慢开始、拥塞避免、快重传和快恢复等机制实现拥塞控制，防止网络过载；通过三次握手建立连接，通过四次挥手释放连接，从而实现可靠的端到端通信。

第 6 章 应用层#

1. 域名系统DNS的作用是什么？#

DNS 是域名系统，它采用分布式数据库结构，主要作用是实现域名到 IP 地址的映射，使用户可以使用容易记忆的域名访问因特网上的主机和服务。

2. 万维网和HTTP协议是什么？#

万维网 WWW 是因特网上以超文本方式组织和访问信息的应用服务系统，用户通过浏览器使用 URL 访问 Web 服务器上的网页资源。HTTP 是超文本传输协议，是 WWW 服务中浏览器和 Web 服务器之间交换信息的主要应用层协议。HTTP 采用客户/服务器方式和请求/响应模式，通常运行在 TCP 之上，默认端口号为 80。

3. 说明电子邮件的传输过程以及 SMTP 协议、POP3 协议、IMAP 协议的使用场合。#

电子邮件发送时，发件人先用用户代理编辑邮件，并通过 SMTP 协议把邮件提交给发件人的邮件服务器；发件人邮件服务器再通过 SMTP 协议把邮件转发到收件人的邮件服务器；收件人的邮件服务器保存邮件，收件人使用用户代理通过 POP3 或 IMAP 协议读取邮件。SMTP 用于发送和转发邮件，POP3 用于从服务器下载邮件到本地，IMAP 用于在服务器上管理和同步邮件，适合多设备访问。
