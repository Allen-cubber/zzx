8086微处理器的存储器寻址方式之所以设计得如此多样，核心目的只有一个：**为了高效地支持高级语言中的各种数据结构（如变量、指针、数组、结构体等），并以最少的指令周期完成数据访问。**

在8086中，物理地址的计算公式固定为：

$Physical Address = Segment Address \times 16 + Effective Address$

（物理地址 = 段地址 $\times$ 16 + 偏移地址/有效地址EA）

这五种存储器寻址方式，本质上就是**计算“有效地址（EA）”的五种不同拼凑方法**。硬件设计者提供这些“积木”，是为了让程序员和编译器能根据不同的应用场景，用最直接的方式定位数据。

以下是这五种寻址方式的原理及其在实际应用中的区别：

### 1. 直接寻址 (Direct Addressing)

- **原理**：有效地址（EA）直接写在指令中。
    
- **格式**：`MOV AX, [1234H]` （将DS:1234H内存处的数据赋给AX）
    
- **设计目的与用法**：
    
    - **对应高级语言**：**全局变量**或**静态变量**。
        
    - **区别**：它的地址在编译时就已经固定死了。适用于访问那些在内存中位置永远不会改变的数据。一旦程序运行，它只能死板地去那个固定的地址取数据。
        

### 2. 寄存器间接寻址 (Register Indirect Addressing)

- **原理**：有效地址存放在指定的寄存器中（仅限 `BX`, `BP`, `SI`, `DI`）。
    
- **格式**：`MOV AX, [BX]`
    
- **设计目的与用法**：
    
    - **对应高级语言**：**指针**。
        
    - **区别**：地址不再是写死的，而是动态的。你可以通过改变寄存器里的值，用同一条指令访问不同的内存单元。这是遍历简单数组或字符串的基础。
        
    - _注_：使用 `BP` 时，默认段寄存器是 `SS`（堆栈段），这对于在栈中传递参数非常重要；其他三个默认是 `DS`（数据段）。
        

### 3. 寄存器相对寻址 (Register Relative Addressing)

- **原理**：有效地址 = 基址/变址寄存器（`BX`, `BP`, `SI`, `DI`） + 8位或16位的位移量（常量）。
    
- **格式**：`MOV AX, [BX+10H]` 或 `MOV AX, 10H[BX]`
    
- **设计目的与用法**：
    
    - **对应高级语言**：**一维数组**或**结构体/记录(Struct)**。
        
    - **区别**：它引入了“基准+偏移”的概念。
        
        - **查表/一维数组**：位移量（常量）作为数组的起始地址，寄存器作为下标（动态改变）。
            
        - **结构体访问**：寄存器指向结构体的首地址，位移量用来选择结构体内的特定成员（比如第一个成员偏移0，第二个成员偏移4）。
            

### 4. 基址变址寻址 (Based Indexed Addressing)

- **原理**：有效地址 = 一个基址寄存器（`BX`或`BP`） + 一个变址寄存器（`SI`或`DI`）。
    
- **格式**：`MOV AX, [BX+SI]`
    
- **设计目的与用法**：
    
    - **对应高级语言**：**二维数组**。
        
    - **区别**：这种方式有两个“变量”。你可以用基址寄存器（比如 `BX`）来定位二维数组的“行”，用变址寄存器（比如 `SI`）来定位“列”。因为两个寄存器的值都可以在循环中动态修改，这使得处理多维数据变得非常灵活。
        

### 5. 相对基址变址寻址 (Relative Based Indexed Addressing)

- **原理**：有效地址 = 基址寄存器 + 变址寄存器 + 8位/16位位移量（常量）。
    
- **格式**：`MOV AX, [BX][SI][10H]`
    
- **设计目的与用法**：
    
    - **对应高级语言**：**结构体数组**或**栈中的二维数组**。
        
    - **区别**：这是8086中最强大、最灵活，也是计算最复杂的寻址方式。它结合了以上所有的优点。
        
        - 举例：你有一个存储学生信息的结构体数组。位移量（常量）可以是某个字段（如“成绩”）在结构体内的偏移；基址寄存器 `BX` 指向数组首地址；变址寄存器 `SI` 用于在不同学生之间切换（相当于 $i \times$ 结构体大小）。
            

---

### 总结对比

| **寻址方式**   | **组成公式 (EA =)** | **灵活性**      | **典型应用场景 (高级语言映射)**                        |
| ---------- | --------------- | ------------ | ------------------------------------------ |
| **直接**     | 常量              | 最低 (写死)      | 访问全局变量、简单标量                                |
| **寄存器间接**  | 寄存器             | 较低 (1个变量)    | C语言指针解引用 (`*p`)                            |
| **寄存器相对**  | 寄存器 + 常量        | 中等 (1变 + 1定) | 访问结构体成员 (`struct.member`) 或一维数组 (`arr[i]`) |
| **基址变址**   | 基址 + 变址         | 较高 (2个变量)    | 访问二维数组 (`arr[i][j]`)                       |
| **相对基址变址** | 基址 + 变址 + 常量    | 最高 (2变 + 1定) | 访问结构体数组的成员 (`arr[i].member`)               |
