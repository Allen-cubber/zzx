你不仅可以去掉这几句，而且就算主程序保留 `PROC FAR` 或者改成 `PROC NEAR`，只要用 `MOV AH, 4CH` 配合 `INT 21H`，都能完美正常退出。

为了让你知其然也知其所以然，我来为你拆解一下这两种写法的区别：

### 1. 教科书上的老式写法（利用堆栈和 PSP 退出）

你红框里圈出的这段特殊代码：

代码段

```
PUSH DS
SUB AX, AX  ; 相当于把 AX 清零
PUSH AX
```

以及配合它在程序末尾的 `RET` 指令，是 **DOS 1.0 时代的遗留产物**。

- **原理：** 当 DOS 加载一个 `.EXE` 程序时，`DS` 寄存器默认指向一个叫做 **PSP（程序段前缀）** 的内存区域。而在 PSP 的最开头（偏移地址 0000H 处），存放着一条 `INT 20H` 指令（用于结束程序）。
    
- **操作：** `PUSH DS` 把 PSP 的段地址压入堆栈；`SUB AX, AX` 和 `PUSH AX` 把偏移地址 `0000H` 压入堆栈。
    
- **为何必须是 `PROC FAR`：** 当程序执行到最后的 `RET` 时，因为是在 `PROC FAR`（远过程）中，汇编器会把它翻译成 `RETF`（远返回）。`RETF` 会从堆栈中弹出刚才压入的 `0000H` 给 `IP`（指令指针），弹出 PSP 的段地址给 `CS`（代码段）。这样，程序就直接跳转到了 PSP 的开头，执行了那条隐藏的 `INT 20H`，从而返回 DOS。
    

### 2. 你提出的现代写法（调用 DOS 21H 中断退出）

代码段

```
; 初始化段寄存器 (你的写法非常完美)
MOV AX, STACK
MOV SS, AX
MOV SP, OFFSET TOP
MOV AX, DATA
MOV DS, AX
MOV ES, AX
...
; 退出程序
MOV AH, 4CH
INT 21H
```

- **原理：** 这是 DOS 2.0 之后引入的系统调用。`AH = 4CH` 代表“带返回码终止进程”。
    
- **优势：** 它不依赖堆栈里预先存放的数据，也不管你是通过远跳转（`FAR`）还是近跳转（`NEAR`）进来的。只要执行到 `INT 21H`，系统就会干脆利落地结束程序并把控制权交还给 DOS。
